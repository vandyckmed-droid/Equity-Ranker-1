Patch the portfolio basket and portfolio weighting system with the following goals:

Core product intent:
- I want manual **stock selection and removal** to remain in place.
- I do **not** want manual portfolio weights now or later.

1. Fix portfolio basket persistence so selections do not disappear or zero out

========================================
1) FIX PORTFOLIO BASKET PERSISTENCE
========================================

Problem:
The portfolio selections keep zeroing out or disappearing. The selected basket must persist reliably.

Requirements:
1. Persist the portfolio basket to localStorage
2. Restore it automatically on app load
3. Keep persistence limited to basket selection only
4. Do not persist manual weights because manual weights should not exist
5. Adding/removing names should update storage immediately and reliably
6. Clearing the basket should clear storage too
7. Duplicates must not be allowed
8. Corrupted or invalid localStorage data must fail safely and reset cleanly
9. Preserve basket order if possible
10. Keep implementation simple, robust, and transparent

Store only what is needed:
- selected tickers
- any minimal metadata needed for safe restoration

Do not:
- create placeholder weights
- restore fake holdings
- introduce UI complexity

========================================
2) REMOVE MANUAL-WEIGHT PLUMBING
========================================

Important:
I do NOT want manual weight entry.
I only want manual stock selection/removal.
All weights must be generated automatically from:
- the current basket constituents
- the selected weighting method
- the current risk/covariance settings

Implement:

2. Make the portfolio holdings state represent a basket of tickers, not user-entered weights

4. Keep weighting methods:

5. Update UI text so it is obvious that weights are system-generated from the basket and method
6. Make the code and request/response shapes clearly reflect:
   - basket in
   - automated weights out

Do not add:
- custom weight mode
- weight editing UI
- stored manual weights

========================================
3) UPGRADE AND REFINE MIN VAR
========================================

Goal:
Refine the current Min Var method into a high-quality institutional implementation that remains simple in the UI but materially better in methodology, stability, and trustworthiness.

Design intent:
- no added visual complexity
- no optimizer dashboard
- no black-box behavior
- strong backend methodology
- trustworthy weights for baskets up to 40 stocks

Non-negotiable rules:
1. Keep Min Var long-only
2. Keep Min Var fully invested before any vol-target overlay
3. Keep all safeguards explicit and auditable
4. Prefer robustness over cleverness
5. Prefer transparent math over exotic complexity

Upgrade requirements:
1. Replace plain raw sample covariance as the default optimizer input
2. Use a robust covariance estimator suitable for small-to-medium baskets
3. Prefer shrinkage covariance as the default
4. Add EWMA support only if useful, but prioritize robustness and stability
5. Add numerical stabilization:
   - diagonal ridge / regularization if needed
   - condition checks
   - safe handling of near-singular matrices
6. Add sensible optimizer constraints:
   - long-only
   - weights sum to 1
   - max single-name cap
7. Prevent pathological concentration
8. Keep fallback logic explicit:
   - if Min Var fails stability or optimization checks, fall back transparently to Inverse Vol
9. Keep outputs easy to audit

Methodology goals:
- stable covariance estimation
- robust optimization for baskets up to 40 names
- reasonable concentration control
- strong numerical behavior
- transparent fallback logic

Choose the best practical constrained Min Var design for this app.
Do not add unnecessary knobs unless clearly valuable.
Defaults should be strong enough that the method works well without user tuning.
