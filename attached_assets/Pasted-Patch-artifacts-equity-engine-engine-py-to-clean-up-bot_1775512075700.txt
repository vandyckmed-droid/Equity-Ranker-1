==================================================
C) QUALITY SLEEVE — MAKE IT MORE DISCIPLINED
==================================================

Current state:
- profitability = roe if available else roa
- margin = gross_margin if available else op_margin
- leverage = -de_ratio
- each pillar is winsorized then z-scored
- available pillars are averaged
- a stock can get quality from only one component

This should be tightened but not overengineered.

New quality construction:

1. Raw descriptors to keep pulling:
- roe
- roa
- gross_margin
- op_margin
- de_ratio

2. Build 3 buckets:

PROFITABILITY BUCKET
- if both roe and roa exist:
  profitability_raw = mean( standardized_roe_input , standardized_roa_input ) after each series is winsorized and z-scored cross-sectionally
- if only one exists:
  use that one
- if neither exists:
  profitability bucket = NaN

MARGIN BUCKET
- if both gross_margin and op_margin exist:
  margin_raw = mean( standardized_gross_margin_input , standardized_op_margin_input )
- if only one exists:
  use that one
- if neither exists:
  margin bucket = NaN

LEVERAGE BUCKET
- leverage bucket = standardized version of -de_ratio
- lower debt is better
- keep the negation
- let winsorization handle outliers rather than adding clever special cases unless absolutely necessary

Standardization order:
- winsorize first, then z-score
- keep this order
- keep winsorization cross-sectional
- keep the current winsor_p parameter unless there is a strong implementation reason to change it

Final quality score:
- quality = mean of available buckets
- but require:
  profitability bucket must exist
  AND at least one of margin or leverage must exist
- otherwise quality = NaN

Reason:
This preserves the cheap/simple structure but prevents a stock from receiving a “quality” score from leverage alone or from a single weak descriptor.
