3-Pillar Institutional Quality / Junk Factor

Replace the current 5-field partial quality sleeve with a strict 3-pillar QualityCore factor designed for robustness, reliability, and broad data coverage. The current implementation relies on ROE, ROA, Gross Margin, Operating Margin, and D/E, allows partial scoring, and produces uneven comparability because missingness is structural, especially for Gross Margin. The new design uses only Profitability, Investment, and Safety. A stock is quality-eligible only if all 3 pillars are present, allowing fallback within a pillar. Quality’s primary role becomes an optional junk filter; if retained in alpha, its default weight should be reduced.

New factor design

Build QualityCore from exactly 3 pillars:
	1.	Profitability
	•	Primary: operating profitability if already computable robustly from current data
	•	Fallback: ROA
	•	Higher is better
	2.	Investment
	•	Use asset growth or similar balance-sheet growth metric from existing fundamentals
	•	Lower is better
	3.	Safety
	•	Use debt/assets or net debt/assets if already available cheaply
	•	Lower is better

Eligibility rules
	•	A stock is quality-eligible only if all 3 pillars are present
	•	Fallback is allowed within a pillar, not across pillars
	•	Do not compute partial quality scores from 1 or 2 pillars
	•	If any pillar is missing, set quality to not available
	•	Excluding ineligible names from quality-based filtering is acceptable

Factor construction
	•	Use only annual or TTM accounting data already available in the system
	•	Add conservative point-in-time / filing-lag handling if cheap from current architecture
	•	Winsorize each pillar cross-sectionally
	•	Z-score each pillar across the eligible universe
	•	Compute:

QualityCore = average(zProfitability, -zInvestment, -zLeverage)

	•	Default quality filter behavior:

	•	If quality remains in alpha, reduce default weight to 5–10% rather than 20%
