Add five portfolio weighting methods to the app, implemented cleanly and consistently from simplest to most advanced.

GOAL

1. Equal Weight
2. Inverse Volatility
3. Signal / Volatility
4. Risk Parity / Equal Risk Contribution
5. Constrained Mean-Variance Optimization

This should be done in a way that is simple, auditable, and easy to compare. Do not overcomplicate the UI.

- Reuse the existing portfolio page and existing risk pipeline where sensible.
- Keep naming mathematically accurate.

- All methods must produce weights that sum to 1 before any vol-targeting overlay.

METHODS TO IMPLEMENT

1) EQUAL WEIGHT
Definition:
- weight_i = 1 / N

Rules:
- simplest baseline
- must always work
- no dependence on signal or covariance

2) INVERSE VOLATILITY
Definition:
- raw_i = 1 / sigma_i
- weight_i = raw_i / sum(raw)

Where:
- sigma_i = annualized volatility from the portfolio risk model diagonal
- use the same volatility engine used elsewhere in the portfolio system where possible

Rules:
- if sigma_i is missing or invalid, handle safely
- add a volatility floor to avoid absurd weights from tiny sigma
- if method fails, fall back safely

3) SIGNAL / VOLATILITY
Definition:
- raw_i = max(signal_i, 0) / sigma_i
- weight_i = raw_i / sum(raw)

Where:
- signal_i should use the portfolio signal already used in the app, aligned with current Alpha / Global Z logic
- clamp or winsorize extreme signals if needed for stability, but do it simply and document it

Rules:
- negative signals should not create negative weights
- add a volatility floor
- if all signals are non-positive, fall back safely

4) RISK PARITY / EQUAL RISK CONTRIBUTION
Goal:
- choose long-only weights so each holding contributes roughly equal portfolio risk

Requirements:
- use covariance matrix, not just standalone vol
- long-only
- fully invested
- deterministic
- robust enough for baskets up to about 40 names
- use a practical numerical method, not a black box
- keep code understandable

Acceptable approach:
- iterative risk budgeting / equal risk contribution approximation
- do not over-engineer

5) CONSTRAINED MEAN-VARIANCE OPTIMIZATION
Goal:
- build a true constrained optimizer using expected return / signal and covariance

Objective:
- maximize risk-adjusted utility or equivalently solve a standard mean-variance problem using:
  signal vector + covariance matrix

Requirements:
- long-only
- sum(weights) = 1
- max position cap
- deterministic
- properly constrained
- not just solve then clip
- use a real constrained optimization approach or a clear projected optimization method
- keep it robust and readable

Important:
- this method should be clearly distinct from pure Min Var
- if the app currently has misleading labels, fix them
